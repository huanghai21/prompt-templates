# 会议总结生成

## 提示词

```
请根据以下会议记录为我生成一份结构清晰的会议总结。总结应包括：会议基本信息（时间、参与者、目的）、主要讨论内容、决定事项、后续行动计划（包括负责人和截止日期）。请以专业的商务语言呈现，重点突出关键决策和重要信息。

会议记录：
{会议记录内容}
```

## 示例

```
请根据以下会议记录为我生成一份结构清晰的会议总结。总结应包括：会议基本信息（时间、参与者、目的）、主要讨论内容、决定事项、后续行动计划（包括负责人和截止日期）。请以专业的商务语言呈现，重点突出关键决策和重要信息。

会议记录：
产品研发周会，3月25日下午2点，线上Zoom会议。参与者：张总（产品负责人）、李工（技术主管）、王设（设计师）、赵工、钱工、孙工（开发团队）、周经理（项目经理）。

张总首先回顾了上周工作：APP新版本功能开发完成80%，但UI测试发现了3个用户体验问题。李工报告了后端API的开发进度，目前遇到的性能瓶颈可能需要重新设计数据库索引。王设展示了新的UI设计稿，与会者对主页布局提出了修改建议。

团队讨论了发布计划，原定4月5日的发布日期可能需要推迟。张总强调市场部已经安排了宣传，推迟将影响营销计划。经过讨论，大家同意追加资源解决问题，尽量按原计划发布。李工将负责数据库优化，预计2天内完成；王设将修改UI设计稿，明天提交修改版；赵工团队承诺加班解决剩余功能开发。

关于新功能需求，张总提出下版本需要增加社交分享功能，请李工评估技术可行性，下周一前给出方案和时间估计。会议最后安排了代码审查，由钱工负责，本周五前完成。

下次会议定于下周一上午10点，主要检查本周任务完成情况和讨论新功能开发计划。
```

## 说明

这个提示词可以帮助你快速将混乱的会议记录整理成结构化的会议总结，突出重点内容和后续行动计划，提高工作效率。
